What’s new

Each build includes the latest release notes, changelog history, and a summary of the changes since the previous version.

Security Improvements (Highlighted)

  • Secrets no longer echoed — API and webhook secrets are never printed back into admin forms. Inputs are left empty and preserved when unchanged.
  • Autoload disabled for secrets — known sensitive options are migrated to autoload = 'no' so secrets are not loaded on every request.
  • Constant fallbacks — admins may define secrets in wp-config.php (e.g. CHPS_STRIPE_SECRET_KEY) to keep them out of the database.
  • Logging minimized — webhook payloads, signatures, and full remote response bodies are no longer written to plugin logs to avoid accidental exposure.
  • Safe UI & server guards — setup wizard includes a transient guard and confirmation notices; save handlers preserve existing secrets unless explicitly replaced.
  • License enforcement — paid-tier restrictions are enforced before deep scans run, preventing unauthorized corporate scans.

These measures reduce risk for production deployments and shared hosting environments. See the upgrade guide for verification steps.
