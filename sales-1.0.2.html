CuredHosting Plugin Suite 1.0.2

Security-focused update: hardening secret storage, safer logging, and tighter license enforcement.

Security Upgrades — What we changed

  • Never echo secrets: API keys and webhook secrets are not shown in admin forms. Fields are blank; saving without entering a value preserves the existing secret.
  • Autoload disabled for secrets: Sensitive options are migrated to autoload = 'no' so they aren't loaded on every request.
  • Define secrets in config: Define CHPS_STRIPE_SECRET_KEY / CHPS_STRIPE_WEBHOOK_SECRET in wp-config.php to keep secrets out of the DB.
  • Minimal logging: Webhook payloads, signatures, and full remote response bodies are no longer written to logs; only safe, high-level messages remain.
  • Server-side guards: The setup wizard has a transient guard to prevent repeated runs; save handlers preserve secrets unless explicitly changed.
  • License/tier enforcement: Paid-tier behavior (link-audit depth and corporate scans) is enforced before running heavy scans.

These changes reduce the risk of accidental exposure on shared hosts and production sites. We recommend defining secrets in wp-config.php and rotating any keys you suspect may have been previously logged.

Why this matters

We prioritized protecting customer secrets and preventing accidental leakages. These changes balance safety and convenience — admins can still rotate keys via the UI, but secrets stay hidden by default.
